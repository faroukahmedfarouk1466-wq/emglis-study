الفصل الأول: مقدمة إلى PySide6

ما هو PySide6؟

PySide6 هي مكتبة Python الرسمية من Qt Company التي تتيح بناء تطبيقات Desktop احترافية تعمل على Windows وmacOS وLinux. تعتمد على إطار عمل Qt6 المكتوب بـ C++ وتوفر binding كامل له في Python.

صدرت PySide6 عام 2021 مع Qt6، وتعتبر البديل المجاني والرسمي لـ PyQt6 من حيث الترخيص.

لماذا PySide6؟

  • ترخيص LGPL مجاني حتى في التطبيقات التجارية
  • دعم رسمي من Qt Company
  • أداء عالي مبني على C++
  • Widgets غنية وجاهزة للاستخدام
  • دعم كامل للـ Multithreading
  • تعمل على المنصات الثلاثة بنفس الكود

مقارنة مع البدائل

متطلبات التثبيت

### Python

يتطلب PySide6 الإصدار Python 3.8 أو أحدث. يُنصح باستخدام Python 3.11+.

# تثبيت عبر pip

الفصل الثاني: أول تطبيق

هيكل تطبيق PySide6

كل تطبيق PySide6 يتكون من ثلاثة مكونات أساسية:

  • QApplication: كائن يُنشأ مرة واحدة ويدير دورة حياة التطبيق
  • QMainWindow أو QWidget: النافذة الرئيسية
  • exec(): حلقة الأحداث الرئيسية

مثال 1: Hello World

import sys
from PySide6.QtWidgets import QApplication, QLabel, QMainWindow
class MainWindow(QMainWindow):
def __init__(self):

super().__init__()

self.setWindowTitle('أول تطبيق PySide6')
self.setMinimumSize(400, 300)

label = QLabel('مرحباً بالعالم!')

self.setCentralWidget(label)

app = QApplication(sys.argv)

window = MainWindow()

window.show()

sys.exit(app.exec())

مثال 2: نافذة مع زر

import sys
from PySide6.QtWidgets import QApplication, QMainWindow, QPushButton, QVBoxLayout, QWidget
class MainWindow(QMainWindow):
def __init__(self):

super().__init__()

self.setWindowTitle('تطبيق مع زر')
# إنشاء widget مركزي

central = QWidget()

layout = QVBoxLayout()

central.setLayout(layout)

self.setCentralWidget(central)
# إنشاء زر

button = QPushButton('اضغط هنا')

button.clicked.connect(self.on_button_click)

layout.addWidget(button)

def on_button_click(self):

print('تم الضغط على الزر!')

app = QApplication(sys.argv)

window = MainWindow()

window.show()

sys.exit(app.exec())

دورة حياة التطبيق

عند تشغيل التطبيق، يمر بالمراحل التالية:

  • إنشاء QApplication وتهيئة Qt
  • إنشاء النافذة الرئيسية
  • استدعاء show() لعرض النافذة
  • دخول حلقة الأحداث عبر exec()
  • انتظار أحداث المستخدم (ضغط، كتابة، تحريك الفأرة...)
  • عند إغلاق النافذة، تنتهي exec() وينتهي التطبيق

ملاحظة: دائماً مرر sys.argv إلى QApplication حتى لو كان التطبيق بسيطاً.

الفصل الرابع: الـ Layouts

ما هي الـ Layouts؟

الـ Layout هو نظام تخطيط يتحكم في كيفية ترتيب الـ Widgets داخل النافذة. بدلاً من تحديد موضع كل Widget يدوياً، تضعها في Layout ويتولى هو ترتيبها تلقائياً وإعادة ترتيبها عند تغيير حجم النافذة.

أنواع الـ Layouts

hbox.addWidget(QPushButton('يمين'))

main_layout.addLayout(vbox)

الفصل الخامس: Signals & Slots

المفهوم

نظام Signals & Slots هو قلب PySide6. الـ Signal هو حدث يُرسَل عند حدوث شيء ما (مثل الضغط على زر)، والـ Slot هو الدالة التي تستجيب لهذا الحدث.

ربط Signal بـ Slot

from PySide6.QtWidgets import QPushButton

btn = QPushButton('اضغط')

# طريقة 1: ربط بدالة مستقلة
def my_func():

print('تم الضغط!')

btn.clicked.connect(my_func)

# طريقة 2: ربط بـ lambda

btn.clicked.connect(lambda: print('ضُغط الزر'))

# طريقة 3: ربط بـ method
class MyWindow(QMainWindow):
def __init__(self):

super().__init__()

btn = QPushButton('زر')

btn.clicked.connect(self.handle_click)

def handle_click(self):

print('تم الاستجابة')

إنشاء Custom Signals

from PySide6.QtCore import QObject, Signal
class DataProcessor(QObject):
# تعريف signals

data_ready = Signal(list) # signal يحمل list

progress = Signal(int) # signal يحمل int

error = Signal(str) # signal يحمل string

finished = Signal() # signal بدون بيانات

def process(self, data):

for i, item in enumerate(data):

# إرسال الـ signal
self.progress.emit(i * 100 // len(data))
self.data_ready.emit(data)
self.finished.emit()

فصل الربط

# قطع الاتصال

btn.clicked.disconnect(my_func)

# قطع كل الاتصالات

btn.clicked.disconnect()

# ربط مؤقت باستخدام lambda (لا يمكن فصله مباشرة)
# الحل: احفظ الـ lambda في متغير

handler = lambda: print('ضُغط')

btn.clicked.connect(handler)

btn.clicked.disconnect(handler)

الفصل السادس: Multithreading

لماذا نحتاج Multithreading؟

تطبيقات Desktop تحتاج أحياناً لتنفيذ عمليات ثقيلة مثل: تحميل ملفات، استدعاء API، معالجة بيانات ضخمة. لو نفذت هذه العمليات في الـ Main Thread، سيتجمد التطبيق ولن يستجيب للمستخدم.

ملاحظة: لا تُعدِّل واجهة المستخدم من thread غير الـ Main Thread. استخدم Signals دائماً للتواصل.

QThread - الطريقة الصحيحة

from PySide6.QtCore import QThread, Signal
from PySide6.QtWidgets import QMainWindow, QPushButton, QProgressBar
class Worker(QThread):

progress = Signal(int)

result = Signal(str)

error = Signal(str)

def __init__(self, data):

super().__init__()

self.data = data
def run(self):  # هذا يعمل في thread منفصل

try:

for i in range(len(self.data)):

# معالجة البيانات
import time; time.sleep(0.1)
self.progress.emit(i + 1)
self.result.emit('تمت المعالجة بنجاح')

except Exception as e:

self.error.emit(str(e))
class MainWindow(QMainWindow):
def __init__(self):

super().__init__()

self.worker = None
def start_task(self):
self.worker = Worker(['a', 'b', 'c', 'd', 'e'])
self.worker.progress.connect(self.update_progress)
self.worker.result.connect(self.on_done)
self.worker.error.connect(self.on_error)
self.worker.start()
def update_progress(self, value):
self.progress_bar.setValue(value)
def on_done(self, msg):

print(msg)

def on_error(self, err):

print('خطأ:', err)

QRunnable مع QThreadPool

from PySide6.QtCore import QRunnable, QThreadPool, Slot
class Task(QRunnable):

@Slot()

def run(self):

print('تعمل في thread منفصل')

import time; time.sleep(2)

print('انتهت المهمة')

# استخدام thread pool

pool = QThreadPool.globalInstance()

pool.setMaxThreadCount(4) # أقصى عدد threads

task = Task()

pool.start(task)

نمط MVC

يُنصح باتباع نمط Model-View في المشاريع الكبيرة لفصل منطق البيانات عن الواجهة:

  • Model: البيانات ومنطق الأعمال — لا يعرف شيئاً عن الواجهة
  • View: عناصر الواجهة فقط — لا تحتوي على منطق
  • Controller: يربط بين Model و View
# model.py - البيانات فقط
class UserModel:
def __init__(self):
self.users = []
def add_user(self, name, email):
self.users.append({'name': name, 'email': email})
def get_all(self):

return self.users

# view.py - الواجهة فقط
class UserView(QWidget):

user_added = Signal(str, str) # name, email

def __init__(self):

super().__init__()

# ... إنشاء الواجهة
def display_users(self, users):
self.table.setRowCount(len(users))

for i, u in enumerate(users):

self.table.setItem(i, 0, QTableWidgetItem(u['name']))
# controller.py - يربط بينهما
class UserController:
def __init__(self):
self.model = UserModel()
self.view = UserView()
self.view.user_added.connect(self.add_user)
def add_user(self, name, email):
self.model.add_user(name, email)
self.view.display_users(self.model.get_all())
